# Debug: Save Layout Not Persisting

## Symptom
User clicks "Save Layout" button, toast appears, but after reload the layout is reset to default.

## How to Debug (Step by Step)

### Step 1: Open Browser Dev Tools
1. Go to http://localhost:5174/
2. Press `F12` to open Developer Tools
3. Go to **Console** tab

### Step 2: Initial Load - Check Logs
When the page first loads, look for:
```
🚀 [App Mount]
  📂 localStorage["life-city-os-save"]: ...
  🏪 Store cityLayout: ...
```

**What to look for:**
- If `localStorage["life-city-os-save"]: NOT FOUND` → Nothing was ever saved (no prior session)
- If it shows objects → A previous save exists
- Check if Store cityLayout matches the saved version

Also look for:
```
🔄 [REHYDRATE] Store hydration
  Rehydrated state: {...}
```

**What to look for:**
- `layoutSize` should match the number of buildings
- `version` should be `1`
- If you see "Version mismatch" → something went wrong with version checking

### Step 3: Make a Change
1. In the game (white rectangle on right):
   - Click a building to select it
   - Drag it to a new position (you'll see ground tile highlight blue)
   - Drop it
2. In browser console, look for:
   ```
   📍 [updateCityLayout] Saving layout to store + localStorage
   ✅ [updateCityLayout] Layout persisted
   ```

**What to look for:**
- Should see these logs automatically when you drop a building
- If NOT appearing → drag/drop not working properly

### Step 4: Click Save Layout Button
1. Click the red **"💾 Save Layout"** button at bottom of Transform Panel (left sidebar)
2. Toast should appear: **"✅ Layout saved!"**
3. In console, look for:
   ```
   💾 [SAVE LAYOUT] User clicked Save
      cityLayout length: [number]
      First 3 objects: ...
      Saved to localStorage
      ✅ Verified: localStorage key "life-city-os-save" has [number] objects
   ```

**What to look for:**
- If toast doesn't appear → button click not working
- If logs don't appear → save handler not executing
- If `❌ ISSUE: localStorage key not found after save!` → localStorage write failed

### Step 5: Verify localStorage
In **Console**, run:
```javascript
JSON.parse(localStorage.getItem('life-city-os-save')).cityLayout.length
```

**What to look for:**
- Should return a number (count of buildings)
- If `null` → localStorage is empty
- If error → localStorage key doesn't exist

### Step 6: Reload Page
1. Press `Cmd+R` (or `Ctrl+R` on Windows)
2. Look at console logs again
3. Check if the buildings appear in the new positions or reset to default

**What to look for:**
- Does `🚀 [App Mount]` show saved objects in localStorage?
- Does the map show your moved buildings or reset to default?

## Possible Issues & Solutions

### Issue 1: localStorage writes but doesn't load on reload
- Persist middleware timing issue
- **Fix**: Wait for `_hydrated` flag (already implemented)

### Issue 2: Version mismatch on reload
- LAYOUT_VERSION changed or localStorage has wrong version
- **Fix**: Check console for "Version mismatch" warnings

### Issue 3: Save button toast appears but logs show error
- localStorage might be disabled or quota exceeded
- **Fix**: Check browser storage settings

### Issue 4: No logs appear at all
- Zustand store not initialized yet when console checked
- **Fix**: Refresh after page fully loads, then open console

## To Report the Issue
1. Clear console (`Ctrl+L` or click clear button)
2. Refresh page
3. Make a change to a building (drag it)
4. Click Save Layout
5. Reload page
6. Copy ALL console output and share

Include:
- What logs appear on initial load?
- Do the update logs appear when dragging?
- Do the save logs appear when clicking Save?
- What does localStorage contain after save?
- Does reload show the saved layout or default?
